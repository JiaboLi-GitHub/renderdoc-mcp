Second public release of `renderdoc-mcp`, focused on live capture workflows, a cleaner architecture, and protocol/docs alignment.

## Highlights

- Added `capture_frame`, which can launch an application with RenderDoc injected, capture a frame, and auto-open the resulting `.rdc`
- Added `renderdoc-cli` live capture support so the same core capture path is available from the command line
- Expanded the MCP surface to 21 tools, including the new live-capture workflow
- Refactored the codebase into clearer `core`, `mcp`, `cli`, and `skill` layers to make future extensions safer

## Improvements

- Updated integration tests to match the current MCP response schema
- Synced English and Chinese README examples with the actual `totalEvents` / `totalDraws` fields
- Refreshed tool descriptions so the release documentation matches runtime behavior

## Notes

- Windows x64 only
- Release artifacts continue to package the RenderDoc v1.36 runtime files
- Verified with the current Release workflow and local `ctest` run before publishing
