Patch release focused on making the Windows release package directly installable in Codex and aligning the published artifacts with the documented project structure.

## Highlights

- Release packages now include both `renderdoc-mcp.exe` and `renderdoc-cli.exe`
- Release packages now bundle the `renderdoc-mcp` skill under `skills/renderdoc-mcp/`
- Added `install-codex.ps1` to install the packaged MCP server, skill, and CLI into Codex with the MCP name fixed to `renderdoc-mcp`

## Improvements

- Switched release packaging to stage files from the CMake install layout so published artifacts match repository install rules
- Added release-package validation in both the packaging script and the GitHub Release workflow
- Updated English and Chinese README instructions for the official GitHub release install path

## Notes

- Windows x64 only
- Codex installs the MCP server under the fixed name `renderdoc-mcp`
- Restart Codex Desktop after running `install-codex.ps1`
