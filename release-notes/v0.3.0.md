# v0.3.0 ‚Äî Phase 2-4: Shader Editing, Diff Engine, Pass Analysis

**Date:** 2026-04-02

## Highlights

- **52 MCP tools** (25 new since v0.2.x)
- Phase 2: Shader hot-editing, mesh/snapshot export, CI assertions (13 tools)
- Phase 3: Dual-capture diff engine with 6 comparison modes (8 tools)
- Phase 4: Pass-level analysis ‚Äî attachments, statistics, dependency DAG, unused RT detection (4 tools)

## Phase 2: Shader Editing, Extended Export, CI Assertions (13 tools)

### Shader Hot-Editing
- `shader_encodings` ‚Äî List supported shader compilation encodings
- `shader_build` ‚Äî Compile shader source code
- `shader_replace` ‚Äî Replace shader at event/stage
- `shader_restore` ‚Äî Restore single shader
- `shader_restore_all` ‚Äî Restore all shaders

### Extended Export
- `export_mesh` ‚Äî Export post-transform vertex data as OBJ or JSON
- `export_snapshot` ‚Äî Export complete draw state (pipeline + shaders + render targets)
- `get_resource_usage` ‚Äî Query resource usage across all events

### CI Assertions
- `assert_pixel` ‚Äî Validate pixel RGBA with tolerance
- `assert_state` ‚Äî Validate pipeline state fields
- `assert_image` ‚Äî Compare PNG images pixel-by-pixel
- `assert_count` ‚Äî Validate resource/event/draw counts
- `assert_clean` ‚Äî Validate no debug messages above severity

## Phase 3: Diff Engine (8 tools)

Dual-capture comparison with LCS-aligned draw matching:

- `diff_open` ‚Äî Open two captures for comparison
- `diff_close` ‚Äî Close diff session
- `diff_summary` ‚Äî High-level comparison (draw/resource/pass counts)
- `diff_draws` ‚Äî Per-draw comparison with LCS alignment
- `diff_resources` ‚Äî Resource list comparison
- `diff_stats` ‚Äî Per-pass statistics delta
- `diff_pipeline` ‚Äî Pipeline state comparison at specific draw
- `diff_framebuffer` ‚Äî Pixel-level framebuffer comparison

### CLI
- `renderdoc-cli diff FILE_A FILE_B [--draws|--resources|--stats|--pipeline MARKER|--framebuffer]`

## Phase 4: Pass-Level Analysis (4 tools)

Hybrid pass enumeration: marker-based passes with synthetic gap-fill for uncovered events.

- `get_pass_attachments` ‚Äî Query color/depth attachments for a render pass
- `get_pass_statistics` ‚Äî Per-pass aggregated statistics (draws, triangles, RT dimensions)
- `get_pass_deps` ‚Äî Inter-pass resource dependency DAG
- `find_unused_targets` ‚Äî Detect render targets written but never consumed (with wave ranking)

### CLI
- `renderdoc-cli <capture.rdc> pass-stats`
- `renderdoc-cli <capture.rdc> pass-deps`
- `renderdoc-cli <capture.rdc> unused-targets`

## Notes

- Windows x64 only
- Requires RenderDoc v1.43+ runtime (bundled in release packages)
