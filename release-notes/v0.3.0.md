# v0.3.0 — Phase 2: Shader Editing, Extended Export, CI Assertions

**Date:** 2026-04-01

## Highlights

- **40 MCP tools** (13 new in Phase 2)
- Shader hot-editing: compile, replace, and restore shaders at runtime
- Mesh export: OBJ/JSON vertex data from draw calls
- Draw snapshots: export complete rendering state in one call
- Resource usage tracking: see how resources are used across events
- CI assertions: automated validation of pixel values, pipeline state, images, counts, and debug messages

## New Tools

### Shader Hot-Editing
- `shader_encodings` — List supported shader compilation encodings
- `shader_build` — Compile shader source code
- `shader_replace` — Replace shader at event/stage
- `shader_restore` — Restore single shader
- `shader_restore_all` — Restore all shaders

### Extended Export
- `export_mesh` — Export post-transform vertex data as OBJ or JSON
- `export_snapshot` — Export complete draw state (pipeline + shaders + render targets)
- `get_resource_usage` — Query resource usage across all events

### CI Assertions
- `assert_pixel` — Validate pixel RGBA with tolerance
- `assert_state` — Validate pipeline state fields
- `assert_image` — Compare PNG images pixel-by-pixel
- `assert_count` — Validate resource/event/draw counts
- `assert_clean` — Validate no debug messages above severity
