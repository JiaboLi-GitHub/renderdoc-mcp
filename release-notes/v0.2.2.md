Patch release focused on making Vulkan live capture use the same packaged RenderDoc runtime for both capture and replay.

## Highlights

- Upgraded the bundled RenderDoc runtime used by official Windows releases to `v1.43`
- Release packages now include `bin/renderdoc.json` alongside `bin/renderdoc.dll`
- `capture_frame` now points Vulkan implicit layer discovery at the packaged runtime directory so live capture does not drift onto a different system RenderDoc version

## Improvements

- Added package validation for the bundled RenderDoc version and Vulkan layer manifest metadata
- Updated local build/install flow so `renderdoc.json` is copied next to the executables during builds and packaged installs
- Updated install-time validation to require the Vulkan layer manifest in official packages

## Notes

- Windows x64 only
- Vulkan layer-path overrides are ignored for elevated target processes by the Vulkan loader, so capture targets should be launched without elevation when possible
